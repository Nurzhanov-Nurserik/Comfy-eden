# SPEC бота (скилл агента `telegram-bot-dev`)

Перед кодом проектируем бота на бумаге: сценарий, какие данные нужны, где хостить. Бот — отдельный компонент, цены даёт он, не сайт.

## Шаги
1. Зафиксируй зависимости: цены стандартных размеров постельного, прайс одноразовых, правила расчёта (объём, скидки, минимум) — §8.6.
2. Опиши сценарий: гейт (имя+телефон) → авто-прайс/КП по фикс-позициям → нестандарт уточняем и → менеджер → отправка каталога.
3. Опиши состояния диалога и хранимые данные (минимум; приватность контактов).
4. Выбери стек (Python aiogram / Node grammY) и хостинг; сохрани как `BOT_SPEC.md`.

## Проверка
- [ ] Гейт «имя+телефон» описан как обязательный.
- [ ] Есть ветка нестандарта → менеджер.
- [ ] Секреты/токен — вне репозитория; зависимости перечислены.

## Пример
Собрали `BOT_SPEC.md`: aiogram + вебхук, который принимает лиды сайта (`CONFIG.leadEndpoint`); PDF КП — через скилл `generate-kp`; стандартные размеры тянутся из прайс-таблицы бота.

Связано: агент `telegram-bot-dev`; скилл `generate-kp`; файлы `SPEC.md` §8, `PLAN.md` фаза «П», `TASK.md` эпик E.
