# QA-приёмка сайта (скилл агента `design-qa`)

Проверяем, что сайт выглядит дорого и работает: вёрстка не едет, motion не раздражает, формы и ссылки живые. Код не правим — отдаём отчёт.

## Шаги
1. Подними превью на `index.html`.
2. Сделай скриншоты на ширинах **375**, **768**, **1280** и проверь консоль на ошибки.
3. Пройди чеклист: вёрстка (нет горизонтального скролла), hero, sticky-сторителлинг `#products`, мобильное меню, motion + `prefers-reduced-motion`, доступность (label, фокус, контраст), ссылки WhatsApp/tel/email/Telegram, валидация формы.
4. Собери отчёт: `[severity] файл:строка — проблема — рекомендация` (severity: blocker/major/minor/nitpick) + вердикт «годится в прод / нужны правки».

## Проверка
- [ ] Нет blocker/major. Нет ошибок в консоли.
- [ ] На 375px нет горизонтального скролла.
- [ ] При reduced-motion контент виден, анимации выключены.

## Пример
После правки hero запустили `qa-review`: на 375px появился горизонтальный скролл из-за `.hero__title`. Находка: `[major] assets/css/styles.css:~520 — заголовок шире вьюпорта — уменьшить нижнюю границу clamp у .hero__title`.
